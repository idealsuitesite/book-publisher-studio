## How this maps to `docs/VISION.md`'s stage progression

```
MVP (v0.1.0 - v0.8.0-alpha) → Beta (v0.9.0-alpha - v0.20.0-beta) → Public Beta → Commercial Launch (v1.0.0) → Enterprise
```

(Boundary shifted with the 2026-07-17 renumbering above, same day as the row above it — same content boundary, Premium UI/UX still the last MVP milestone, just under its corrected version number. **Beta's upper bound moved from `v0.11.0-beta` to `v0.12.0-beta` on 2026-07-18**, when Sprint 8 took the `v0.9.0-alpha` slot for Publishing Engine and every later row shifted one version — the MVP boundary itself is unchanged, and `v0.9.0-alpha` is now the first *released* Beta-stage version. **Moved again the same day, to `v0.20.0-beta`, by ADR-0039's strategic reprioritization** — Sprints 9–17 now target product completeness (UI, UX, Workspace, Autosave, Performance, Collaboration, Cloud Sync, Licensing, Telemetry) ahead of Editorial AI Engine and Plugin System, which move to `v0.19.0-beta`/`v0.20.0-beta`. The Beta stage is longer, not different: the MVP boundary and the v1.0.0 target are both unchanged.)

## Updating this file

- Add a row when a new version is planned, even before it's built — mirrors how ADR-0012–0016 were written before Sprint 2 code
- Move a row's Status to ✅ Released only after the tag is actually pushed (matches the discipline established for `v0.2.0-alpha`: tag, then Release Notes, then this index)
- Never delete a row — like `docs/DECISIONS.md`, this is meant to read as history in a few years, not just current status
